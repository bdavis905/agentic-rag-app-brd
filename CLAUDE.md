# CLAUDE.md

Agentic RAG app with chat (default) and document ingestion interfaces. Config via env vars, no admin UI.

## Stack
- Frontend: React + Vite + Tailwind + shadcn/ui
- Backend: Convex (schema, functions, file storage, vector search, HTTP actions)
- Auth: Clerk (JWT-based, integrated with Convex)
- LLM: OpenAI (embeddings, metadata), OpenRouter (chat completions)
- Hosting: Vercel (frontend) + Convex Cloud (backend)

## Architecture
- `frontend/convex/` contains all backend logic (queries, mutations, actions)
- `frontend/src/` is the React app
- Frontend talks to Convex directly via hooks (`useQuery`, `useMutation`)
- Chat streaming uses an HTTP Action (`convex/http.ts`) with SSE
- Auth tokens come from Clerk, validated by Convex via JWT
- No Python, no FastAPI, no Supabase — everything runs on Convex + Clerk

## Rules
- No LangChain, no LangGraph — raw SDK calls only
- Stream chat responses via SSE (HTTP Action)
- Convex queries auto-update (no manual refetch needed)
- All Convex functions must check `ctx.auth.getUserIdentity()` and filter by `identity.subject`
- API keys are Convex server-side env vars — never exposed to frontend
- Ingestion is manual file upload only — no connectors or automated pipelines

## Planning
- Save all plans to `.agent/plans/` folder
- Naming convention: `{sequence}.{plan-name}.md` (e.g., `1.auth-setup.md`, `2.document-ingestion.md`)
- Plans should be detailed enough to execute without ambiguity
- Each task in the plan must include at least one validation test to verify it works
- Include a complexity indicator at the top of each plan:
  - **Simple** — Single-pass executable, low risk
  - **Medium** — May need iteration, some complexity
  - **Complex** — Break into sub-plans before executing

## Development Flow
1. **Plan** — Create a detailed plan and save it to `.agent/plans/`
2. **Build** — Execute the plan to implement the feature
3. **Validate** — Test and verify the implementation works correctly
4. **Iterate** — Fix any issues found during validation

## Running Locally

Start both services (two terminals):

```bash
# Terminal 1: Convex dev server (watches for changes, deploys functions)
cd frontend && npx convex dev

# Terminal 2: Vite dev server (frontend)
cd frontend && npm run dev
```

- **Frontend:** http://localhost:5173
- **Convex Dashboard:** https://dashboard.convex.dev

## Convex Patterns

- **Queries:** `useQuery(api.module.functionName, args)` — returns `undefined` while loading
- **Mutations:** `useMutation(api.module.functionName)` — returns callable function
- **File upload:** `generateUploadUrl` → POST to URL → create document record
- **Auth:** `ctx.auth.getUserIdentity()` in every public function
- **Reactivity:** Convex queries auto-update — no manual refetch
- **Node actions:** Use `"use node"` for actions needing npm packages (OpenAI, pdf-parse, mammoth)
- **HTTP Actions:** Used for SSE streaming (chat endpoint in `convex/http.ts`)
