# Module 8: Sub-Agents Implementation Plan

**Complexity:** 🔴 Complex - Multiple phases, changes across backend and frontend

## Overview

Add hierarchical agent architecture where the main agent can spawn isolated sub-agents for full-document analysis. When a user asks to analyze an entire document (summarize, review, extract key points), the main agent delegates to a sub-agent with:
- Isolated message context with full document content
- Document-specific analysis capabilities
- Streaming reasoning visible to the user

## Design Decisions

1. **Trigger Mechanism:** New `analyze_document` tool that the main agent calls explicitly
2. **Sub-Agent Scope:** Document-only context, no external tools (search, web, SQL)
3. **UI Display:** Collapsible panel showing sub-agent activity during streaming
4. **No DB Changes:** Sub-agent events stored in memory only (traced via LangSmith)

## Wave 3: Integration Layer (Parallel)

### Task 1.3: Sub-Agent Service
**Depends on:** 1.1, 1.2
**Create:** `backend/app/services/sub_agent_service.py`

- `run_sub_agent(document_id, user_id, user_query)` - AsyncGenerator yielding SSE events
- System prompt focused on document analysis
- No external tools - just content analysis
- Events yielded:
  - `sub_agent_start` - `{document_id, filename, token_count}`
  - `sub_agent_reasoning` - `{content}` (streaming text)
  - `sub_agent_complete` - `{result}`
  - `sub_agent_error` - `{error}`

**Validate:** Call directly, verify events stream correctly

### Task 3.5: Update ChatView
**Depends on:** 3.2, 3.3, 3.4
**Modify:** `frontend/src/components/chat/ChatView.tsx`

Add state:
- `toolCalls: ToolCallInfo[]`
- `subAgent: SubAgentState | null`

In `sendMessage` call, add callbacks:
- `onToolCallStart` - add to toolCalls with 'running' status
- `onSubAgentStart` - set subAgent active with filename
- `onSubAgentReasoning` - append to subAgent.reasoning
- `onSubAgentComplete` - set subAgent status to 'completed'
- `onDone` - clear toolCalls and subAgent state

Render (between waiting indicator and streaming content):
- Map toolCalls to `<ToolCallIndicator>` components
- Render `<SubAgentPanel>` when subAgent is active

**Validate:** Frontend compiles, UI renders without errors
