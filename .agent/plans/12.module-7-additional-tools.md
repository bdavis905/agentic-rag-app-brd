# Module 7: Additional Tools - Implementation Plan

## Complexity: ⚠️ **Medium** - Multiple new tools with established patterns. Database-level security for SQL access.

---

## Overview

**From PRD:**
- Text-to-SQL tool (query structured data)
- Web search fallback (when docs don't have the answer)
- Multi-tool agents, routing between structured/unstructured data
- Graceful fallbacks, attribution for trust

---

## Design Decisions

### 1. Text-to-SQL: Raw SQL with Database-Level Security

**Approach:** Allow LLM to generate raw SQL, but enforce security at the Postgres level using a restricted database role.

**Security Model:**
- Create a dedicated Postgres role (`sql_agent_reader`) with minimal privileges
- Grant `SELECT` only on a specific sample table (`sales_data`)
- No access to other tables (documents, chunks, settings, auth, etc.)
- Any INSERT, UPDATE, DELETE, or unauthorized table access fails at database level

**Sample Table:** `sales_data` - Pre-populated with sample order/transaction data for querying.

**Why This Approach:**
- Clean security boundary enforced by Postgres itself
- LLM can generate any SQL - flexibility for complex queries
- Educational - demonstrates proper database security (least privilege principle)
- No brittle SQL parsing/validation in application code

### 2. Web Search: Tavily API

**Why Tavily:**
- Purpose-built for AI (returns extracted content, not just URLs)
- Simple API, clean response structure
- No complex parsing needed

**Configuration:** Optional - gracefully disabled if not configured.

DO $$
BEGIN
  IF NOT EXISTS (SELECT FROM pg_roles WHERE rolname = 'sql_agent_reader') THEN
    CREATE ROLE sql_agent_reader NOLOGIN;
  END IF;
END $$;

GRANT USAGE ON SCHEMA public TO sql_agent_reader;
GRANT SELECT ON sales_data TO sql_agent_reader;

### Phase 2: New Services

**Create:** `backend/app/services/sql_agent_service.py`
- `execute_sql_query(sql: str)` - Execute SQL using restricted role
- Uses Supabase connection with `SET ROLE sql_agent_reader` for query
- Returns results as formatted string for LLM context
- Catches and returns permission errors gracefully

**Create:** `backend/app/services/web_search_service.py`
- `get_web_search_settings()` - Read config from global_settings
- `web_search(query, max_results)` - Execute search via Tavily API

## Security Model

| Attempted Action | Result |
|------------------|--------|
| `SELECT * FROM sales_data` | ✅ Success |
| `SELECT * FROM documents` | ❌ Permission denied |
| `DELETE FROM sales_data` | ❌ Permission denied |
| `DROP TABLE sales_data` | ❌ Permission denied |
| `INSERT INTO sales_data ...` | ❌ Permission denied |

The restricted `sql_agent_reader` role has ONLY `SELECT` on `sales_data`. All other operations fail at the Postgres level.
