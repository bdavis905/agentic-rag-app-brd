# Module 6: Hybrid Search & Reranking

**Complexity:** ⚠️ Medium - Multiple services and a migration, but well-scoped with clear integration points.

## Overview

Replace pure vector search with a hybrid search pipeline: **keyword (FTS) + vector → RRF fusion → optional reranking**. Reranking supports both Cohere's public API and local/self-hosted models (e.g., Hugging Face TEI) via configurable `base_url`. The existing `search_documents` function signature stays backward-compatible; hybrid becomes the new default.

## Task 1: Database Migration

**Create:** `supabase/migrations/20260126000000_hybrid_search.sql`

- Add `fts tsvector` column to `chunks` table
- Backfill existing chunks: `UPDATE chunks SET fts = to_tsvector('english', content)`
- GIN index on `fts` column
- Trigger to auto-populate `fts` on INSERT/UPDATE of `content`
- New RPC: `keyword_search_chunks(p_query, p_match_count, p_user_id, p_metadata_filter)` using `plainto_tsquery` + `ts_rank_cd`
- Add columns to `global_settings`: `rerank_model` (TEXT, default 'rerank-v3.5'), `rerank_base_url` (TEXT, default 'https://api.cohere.com/v2'), `rerank_api_key` (TEXT), `rerank_top_n` (INTEGER, default 5)
  - For Cohere: base_url = `https://api.cohere.com/v2`, model = `rerank-v3.5`
  - For local (e.g., TEI): base_url = `http://localhost:8080`, model = local model name, api_key optional
  - Service always POSTs to `{base_url}/rerank` with Cohere-compatible request format

**Validate:** Run migration, confirm `SELECT fts FROM chunks LIMIT 1` returns non-null tsvector.

## Task 2: Reranking Service

**Create:** `backend/app/services/rerank_service.py`

- `get_rerank_settings()` → reads `rerank_model`, `rerank_base_url`, `rerank_api_key`, `rerank_top_n` from `global_settings`, decrypts API key
  - Returns None if no base_url configured (reranking disabled)
- `rerank_documents(query, documents, top_n)` → POSTs to `{base_url}/rerank` via `httpx.AsyncClient`
  - Request body: `{"model": model, "query": query, "documents": texts, "top_n": n}` (Cohere-compatible format)
  - Auth header: `Bearer {api_key}` (skipped if api_key is empty, for local models that don't need auth)
- Graceful degradation: returns input unchanged if unconfigured or on error
- Adds `rerank_score` field to each returned result

No new dependencies needed (`httpx` already in requirements).

## Task 3: Hybrid Retrieval Service

**Modify:** `backend/app/services/retrieval_service.py`

Replace current implementation with:

- `vector_search(query, user_id, top_k, threshold, metadata_filter)` → existing `match_chunks` RPC
- `keyword_search(query, user_id, top_k, metadata_filter)` → new `keyword_search_chunks` RPC
- `reciprocal_rank_fusion(result_lists, k=60)` → merges by chunk ID, scores with `1/(k+rank)`
- `search_documents(query, user_id, top_k=5, threshold=0.3, metadata_filter=None, search_mode="hybrid")`
  - `"hybrid"`: runs both searches, fuses with RRF, then reranks
  - `"vector"`: pure vector (original behavior)
  - `"keyword"`: pure FTS
  - Fetches `4 * top_k` candidates before fusion/reranking, then trims
