# Phase 5: Chat — Streaming, Tools, Sub-Agents

**Prerequisite:** Read `.agent/plans/15.convex-migration-progress.md` first for context.
**Depends on:** Phase 4 complete (search works)

**Complexity:** 🔴 Complex — consider splitting into 5a (basic chat) and 5b (tools + sub-agents)

## Goal
Full chat with streaming, tool-calling loop, sub-agents. The core RAG experience.

## Reference Files (Python originals)
- `backend/app/services/llm_service.py` — system prompt, tool definitions, streaming
- `backend/app/services/chat_service.py` — message persistence
- `backend/app/services/tool_executor.py` — tool dispatch
- `backend/app/services/sub_agent_service.py` — deep analysis sub-agent
- `backend/app/services/explorer_agent_service.py` — KB exploration agent
- `backend/app/services/folder_navigation.py` — ls, tree, grep, read tools
- `backend/app/services/web_search_service.py` — Tavily web search

## Sub-phase 5a: Basic Chat with Streaming

### 1. Install @convex-dev/agents
```bash
npm install @convex-dev/agents
```

### 2. Register agent component
- Create/update `convex/convex.config.ts` with agent component registration

### 3. Create `convex/chat/agent.ts`
- Define agent with system prompt (port from `llm_service.py`)
- Configure OpenRouter model (or OpenAI)
- Register tools: `search_documents` first

### 4. Create `convex/chat/tools.ts`
- `search_documents` tool — calls hybridSearch action
- Returns formatted search results with source citations

### 5. Create `convex/http.ts`
- HTTP Action for SSE streaming chat endpoint
- Route: POST `/api/chat`
- Streams: text deltas, tool calls, done events

### 6. Update frontend chat components
- `ChatPage.tsx` — thread list from agent component queries
- `ChatView.tsx` — connect to streaming endpoint
- `ThreadList.tsx` — Convex queries for thread CRUD
- Keep existing UI components (ToolCallIndicator, ThinkingPanel, etc.)

### 7. Auto-title generation
- On first message, use LLM to generate a short title
- Update thread record

## Sub-phase 5b: Advanced Tools + Sub-Agents

### 8. Create `convex/navigation/tools.ts`
- Port ls, tree, grep, read from `folder_navigation.py`
- These let the chat agent browse the knowledge base folder structure

### 9. Create `convex/chat/subAgents.ts`
- `analyze_document` — deep analysis sub-agent (port from sub_agent_service.py)
- `explore_knowledge_base` — multi-round exploration (port from explorer_agent_service.py)

### 10. Add web_search tool
- Tavily API call via fetch
- Uses `TAVILY_API_KEY` env var

### 11. Wire sub-agent events to frontend
- Sub-agent start/reasoning/complete events
- Explorer tool calls display
- Integrate with existing SubAgentPanel and StepsPanel components

## Validation
- Create thread → send message → see streaming response
- Tool calls displayed in UI (search_documents)
- Search returns relevant chunks from uploaded documents
- Sub-agent analysis works on individual documents
- Explorer agent navigates knowledge base
- Thread title auto-generates
- Web search works (if configured)

## Key: @convex-dev/agents Architecture
The agent library handles:
- Thread and message persistence (no custom tables needed)
- Tool-calling loop (automatic)
- Streaming via HTTP Actions
- Context window management

Frontend queries the agent component's tables directly for thread list and message history.
