  - Custom SSE stream parser (callOpenAIStreaming)

  - explore_knowledge_base sub-agent (multi-round KB exploration, max 8 rounds)
  - Title generation on first message
  - CORS handling (preflight + headers)
  - Tool-calling loop (max 10 rounds)
- Updated `convex/documents/internals.ts` — added `fullText` to updateDocumentStatus
- Updated `convex/documents/actions.ts` — stores fullText (truncated at 800K chars) during ingestion
- Updated `convex/_generated/api.ts` — added chat and navigation function references

**Frontend:**
- Rewrote `src/lib/api.ts` — real SSE `sendMessage` connecting to HTTP Action, with auth token, SSE event parsing for all 16 event types. Removed thread/message stubs.
- Rewrote `src/components/chat/ThreadList.tsx` — uses `useQuery`/`useMutation` from Convex (reactive thread list, no more forwardRef/imperative handle)
- Rewrote `src/pages/ChatPage.tsx` — uses Convex `useMutation` for thread creation, removed ThreadListRef
- Rewrote `src/components/chat/ChatView.tsx` — uses `useQuery` for initial message loading, Clerk `getToken` for HTTP Action auth, one-time initialization pattern (useQuery for load, local state for streaming)
- Updated `src/hooks/useAuth.ts` — exposed `getToken` for Clerk JWT retrieval

**Architecture decisions:**
- V8 runtime for HTTP Action (no "use node") — raw fetch to OpenAI instead of SDK
- SSE format: `data: {"type": "xxx", ...}\n\n` for all events
- fullText stored on documents table for navigation tools (read/grep)
- Navigation tools as internal queries returning formatted strings
- Sub-agents run within HTTP Action handler (not separate actions)
- ThreadList is fully reactive via Convex subscription (no imperative updates needed)
- Message loading uses useQuery one-time initialization + local state for streaming

## Important: Before Next Session

The Convex project needs to be connected. Run:
```bash
cd frontend
npx convex dev
```
This will:
1. Prompt to link to the existing "agentic-rag-app-genesis" project
2. Generate proper `convex/_generated/` files (overwriting stubs)
3. Deploy the schema + functions to Convex Cloud

You also need to set env vars:
```bash
# In Convex dashboard
npx convex env set CLERK_JWT_ISSUER_DOMAIN "https://your-clerk-domain.clerk.accounts.dev"
npx convex env set OPENAI_API_KEY "sk-..."            # Required: LLM + title generation
npx convex env set EMBEDDING_API_KEY "sk-..."
npx convex env set EMBEDDING_BASE_URL "https://api.openai.com/v1"
npx convex env set LLM_BASE_URL "https://openrouter.ai/api/v1"  # Optional: OpenRouter
npx convex env set TAVILY_API_KEY "tvly-..."           # Optional: web search tool

# In frontend/.env
VITE_CONVEX_URL=https://your-project.convex.cloud
VITE_CLERK_PUBLISHABLE_KEY=pk_test_...
```
