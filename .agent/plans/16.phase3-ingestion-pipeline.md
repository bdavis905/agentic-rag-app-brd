# Phase 3: Ingestion Pipeline

**Prerequisite:** Read `.agent/plans/15.convex-migration-progress.md` first for context.

**Complexity:** 🔴 Complex

## Goal
Full document processing — parse, chunk, embed, store vectors. Replaces Python Docling + embedding pipeline.

## Pre-session Setup
- `npx convex dev` must have been run (generates `convex/_generated/`)
- `OPENAI_API_KEY` set in Convex env vars
- `EMBEDDING_API_KEY` and `EMBEDDING_BASE_URL` set in Convex env vars

## Reference Files (Python originals to port)
- `backend/app/services/chunking_service.py` — recursive character splitter algorithm
- `backend/app/services/embedding_service.py` — OpenAI-compatible embedding calls
- `backend/app/services/ingestion_service.py` — orchestration pipeline
- `backend/app/services/metadata_service.py` — LLM-based metadata extraction

## Steps

### 1. Create `convex/ingestion/parsing.ts`
- `"use node"` directive (required for pdf-parse and mammoth)
- `extractText(storageId)` — downloads file from Convex storage, extracts text
- PDF: use `pdf-parse` package
- DOCX: use `mammoth` package (extractRawText)
- TXT/MD: direct UTF-8 decode
- Return extracted text string

### 2. Create `convex/ingestion/chunking.ts`
- Port the recursive character splitter from `chunking_service.py`
- Parameters: `chunkSize` (default 1000), `chunkOverlap` (default 200)
- Separators: `["\n\n", "\n", " ", ""]`
- Algorithm: try to split on largest separator first, recursively split if chunks are too large
- Return array of `{ content: string, chunkIndex: number }`

### 3. Create `convex/ingestion/embedding.ts`
- `"use node"` directive
- `getEmbeddings(texts: string[])` — calls OpenAI-compatible embedding API
- Batch size: 250 texts per API call
- Uses `EMBEDDING_API_KEY`, `EMBEDDING_BASE_URL`, embedding model from settings
- Returns `number[][]` (array of embedding vectors)

### 4. Create `convex/ingestion/metadata.ts`
- `"use node"` directive
- `extractMetadata(text: string, schema: MetadataFieldDefinition[])` — LLM-based extraction
- Uses OpenAI structured output (JSON mode)
- Port the system prompt from `metadata_service.py`
- Returns `Record<string, any>` matching the schema

### 5. Create `convex/documents/actions.ts`
- `"use node"` directive
- `processDocument` internalAction — the orchestration pipeline:
  1. Update status → "processing"
  2. Download file from Convex storage
  3. Extract text (parsing.ts)
  4. Chunk text (chunking.ts)
  5. Generate embeddings (embedding.ts, batched)
  6. Store chunks with embeddings in `chunks` table
  7. Optionally extract metadata (metadata.ts)
  8. Update document status → "completed" with chunkCount
  9. On error: update status → "failed" with errorMessage

### 6. Wire up scheduling
- In `convex/documents/mutations.ts: create()`, add:
  ```typescript
  await ctx.scheduler.runAfter(0, internal.documents.actions.processDocument, {
    documentId: docId
  });
  ```
- Remove the TODO comments

### 7. Content hash deduplication
- SHA-256 computed client-side (already done in DocumentUpload.tsx)
- Check in `create` mutation: same filename + same hash = skip
- Same filename + different hash = update and re-process

## Validation
- Upload a PDF → status goes pending → processing → completed → chunkCount > 0
- Upload same file again → skipped (content hash match)
- Upload a DOCX → completes successfully
- Upload a TXT/MD → completes successfully
- Upload unsupported type → appropriate error

## Key Constants to Port
From the Python code:
- Default chunk size: 1000 chars
- Default chunk overlap: 200 chars
- Embedding batch size: 250
- Embedding dimensions: 1536 (text-embedding-3-small)
