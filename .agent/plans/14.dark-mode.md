# Dark Mode Implementation Plan

**Complexity:** ✅ Simple - Single-pass executable, low risk

## Overview

Add dark mode toggle to the frontend. The infrastructure is already in place:
- Tailwind configured with `darkMode: ["class"]`
- CSS variables for light/dark themes defined in `index.css`
- All shadcn/ui components use semantic color variables that auto-adapt

## Implementation Tasks

### Task 1: Create Theme Hook
**File:** `frontend/src/hooks/useTheme.ts` (NEW)

Create a custom hook to manage theme state with localStorage persistence:
- State: `'light' | 'dark' | 'system'`
- Initialize from localStorage or system preference
- Persist changes to localStorage
- Apply/remove `dark` class on `document.documentElement`

```typescript
// Key functions:
// - getInitialTheme(): Check localStorage, fallback to system preference
// - setTheme(theme): Update state, localStorage, and DOM
// - Toggle between light/dark (or cycle light/dark/system)
```

### Task 2: Add Toggle to UserMenu
**File:** `frontend/src/components/UserMenu.tsx`

Add a theme toggle button in the user menu popover:
- Import useTheme hook
- Add Moon/Sun icon button (lucide-react icons)
- Show current theme state
- Click to toggle between light/dark

Layout: Add toggle as first item in the menu, before Settings and Log Out.

### Task 3: Initialize Theme in App Root
**File:** `frontend/src/App.tsx`

Call useTheme at the app root to ensure theme is applied on initial load:
- Import and call useTheme() at top of App component
- This ensures the hook runs and applies the saved theme class
